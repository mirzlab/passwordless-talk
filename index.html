Les problématiques du mot de passe #2



  • Les utilisateurs réutilisent le même mot de passe entre plusieurs services (professionnels et personnels)


  • Les politiques de mot de passe en entreprise génèrent beaucoup de frustration


  • Les rotations de mot de passe sont inefficaces, les utilisateurs remplaçant simplement un nouveau caractère dans leur ancien mot de passe


  • Les attaques par phishing ou bruteforce sont très courantes pour le vol de d’identifiants

L'alliance FIDO

Un consortium d’entreprise (Apple, Amazon, Google, Intel, Microsoft, …) …
… dont l’objectif est de définir les nouveaux standards et promouvoir les technologies « phishing-resitant » permettant une alternative aux mots de passe.

FIDO2, un ensemble de standard

Lorsqu’on accède à une application web, en remplacement du mot de passe, FIDO2 repose sur des « Authenticators »
Une authentification FIDO2 repose sur 2 protocoles : CTAP et WebAuthn

WebAuthn



  • WebAuthn introduit des nouvelles fonctions Javascript permettant aux navigateurs de gérer l’enregistrement et l’authentification d’un utilisateur à travers un Authenticator pour un site web


  • Les paramètres fournis à ces fonctions permettent de «customiser» la cinématique FIDO2 (quel type d’Authenticator souhaité (interne vs externe), quel algorithme de signature utiliser, …)


  • Ces nouvelles fonctions sont aujourd'hui largement supportées par les navigateurs (Chrome, Firefox, Edge, Safari, ...)
